I Use Vim

23-12-2016
Tech
Tools

All people have their favourite tools to write on. Such as Sublime, notepad++, Ms. Word, OneNote, iA Writer, WriteMonkey, and so on. But let’s face it, you can’t code at Ms. Word, OneNote, iA Writer and WriteMonkey. So as a coder, you always prefer an editor that can do everything for text object. Then finally you have notepad, notepad++, sublime, atom, textmate, or … Vim.

First time I hear about Vim was when I did my assignment using my friend’s laptop. He is a Vim user, he said Vim boost productivity. I don’t really care then since I prefer code at IDE than at text editor. I use text editor mainly to do some HTML or CSS. And yea, I used notepad++, just like all my friend at the college.

But still, his word made me curious, so I try Vim. At first, I really use Vim because it looks cool (seriously). I don’t really care about productivity I gain by using Vim. In the end, I go back to VS because I mainly code C#.

But some of Vim’s feature already engraved in my hand. I can’t leave HJKL’ navigation command and modal editing. Since I prefer typing more than moving my mouse, those features are really nice. Finally, I use VsVim at my VS to accommodate my need.

I completely come back to Vim and make it my main editor last year. After completely fell in love with markdown. And I don’t like using any other editor that makes me leaves home row often. Vim is very comfortable for me.

Right now, I still do the same as before, code light file that doesn’t need compiling at Vim, do most of my writing at Vim, and code C# at VS. I feel fine with that. And this year, I start to configuring my Vim to accommodate how I use it – Yea, I was using stock Vim.

At the first, I use other people’s dotfile. I stumbled upon spf13’s vim configuration. Which lead me to lot of things Vim can become. Code completion, status bar, integrated git service, and solarized. But one thing that I realise after using his config is that I can’t freely customise the Vim to match my usage. And yes, at that time, I don’t understand how to configure Vim at all. VimL? Just heard it; separate .vimrc file? WTH is that? And so many things that make me realise I don’t even understand how my beloved text editor works.

So I started to explore Vim’s configuration by learning spf13’s vim config. And finally, I can make my own configuration, not quite fancy, but already satisfying my current need. You can find my dotfiles here.

I am happy with Vim for text object editing. But I still favour Visual Studio to code C# since … you know … IDE stuff. Complete feature gear for development. But for configuration file like .yaml; markup language like HTML and Markdown; and any other text object where all you need is a text editor, I goes to Vim. Vim is comfortable for my hand who hates to leave keyboard’s home row. I even use VsVim at Visual Studio. Since I seem to can’t let modal function and navigational key go away.

After all, text editor is personal preference. No one can dictate anyone, because our definition of comfort and productivity are different. I am happy with Vim, my friend like VSCode better. Some of my friend still using Notepad++, and lot more using SublimeText2. It is not bad. Since I am more into typing than moving mouse, Vim is definitely my Ace. I was using notepad++, Sublime Text, atom, and bracket. But the one that I really comfortable in using is Vim.
